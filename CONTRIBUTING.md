# Contributing to SG-LegalBench

SG-LegalBench grows like LegalBench (Guha et al., NeurIPS 2023): community
contributors propose new tasks, the maintainers review them against a
fixed methodology bar, and the corpus expands without any single
author having to be a credentialed Singapore lawyer.

This document is how a new task gets added.

## Contribution categories

| Kind | Example | Process |
|---|---|---|
| **New task** | "SGLB-NN Tax-Ruling-QA on IRAS published rulings" | Open a tracking issue, then a PR per §"Adding a new task" |
| **New dataset items** | More PDPC decisions, EmA/PC/ROC sections, etc. | Re-run the relevant ingestion script (`make ingest-pdpc`, `make ingest-sso SSO_CODE=…`) and open a PR with the updated JSONL splits + a one-line changelog |
| **Errata** | A wrong gold label, a malformed citation | PR against `data/sglb_NN/errata.md` (one row per fix; we apply errata at the next minor release with attribution) |
| **Scorer fix** | Edge-case in `_normalise_section_citation` | PR with a failing test + the fix; bumps the dataset version |
| **Adapter** | New public-source adapter (e.g. AustLII SG section) | Implement under `backend/api/adapters/public/`; add fixtures + tests; do **not** add anything paywalled or session-cookie-authenticated |

## The methodology bar (non-negotiable)

Every task must satisfy `docs/coverage-matrix.md` §4. In short:

1. **Mechanical label extraction only.** The gold labels come from a
   regulator or court output by a deterministic extraction rule. Author
   legal judgement is not a label source. If you cannot point to the
   regulator URL + the extraction script that produced the label, the
   task is not in scope.
2. **Public-domain source only.** Benchmark items must trace to a
   freely-accessible SG legal source (SSO, PDPC, MOM, IRAS, AGC,
   eLitigation public pages, CommonLII, Hansard). Paywalled sources
   (LawNet, Practical Law) are usable in the copilot via user-supplied
   credentials but **never** in the benchmark.
3. **Strong evaluators only in `--strict` mode.** Keyword-presence,
   length-threshold, and "contains any of these tokens" evaluators are
   acceptable for smoke tests but rejected by `--strict`. New
   evaluators must be rule-based, deterministic, or LLM-judge with
   disclosed ≥3-judge consensus + reported inter-judge κ.
4. **Post-cutoff held-out split.** Each task ships a held-out split
   from material published after the contamination cutoff (currently
   2026-Q1). Leaderboard rows report both training-era and post-cutoff
   scores.
5. **Receipt provenance.** Any LLM-backed run must emit a reproducibility
   receipt with `prompt_version`, `prompt_sha`, `provider_label`,
   `max_tokens`. The harness emits these automatically when you register
   the runner via `benchmark.llm_runner.register_llm_task(...)`.

## Adding a new task

This is the canonical flow. SGLB-NN means the next free task ID.

### 1. File a tracking issue

Title: `P3: SGLB-NN <Short-Name> — <one-line description>`.
Body: copy the spec skeleton from `docs/sglb_specs/README.md` §
"Per-spec sections" and fill in **Capability**, **Literature anchor**,
**Input contract**, **Output contract**, **Scoring**, **Source
provenance** (named adapter + fields), **Limitations**, **v0.1/v0.2
stratification**.

A maintainer will respond with a yes/no in 14 days. We say no to:
- Tasks that require author legal judgement at label time.
- Tasks scoped to a paywalled corpus.
- Tasks that duplicate an existing capability without adding a
  difficulty-axis cell (`docs/coverage-matrix.md` §3).

### 2. Open a PR with the following layout

```
docs/sglb_specs/SGLB-NN.md                            # the spec
backend/benchmark/dataset_builders/sglb_NN.py         # mechanical extraction
backend/benchmark/tasks/sglb_NN.py                    # oracle runner + register_task
backend/benchmark/tasks/__init__.py                   # +1 import line
backend/benchmark/llm_runner.py                       # +1 prompt builder
backend/benchmark/evaluators.py                       # only if a new scorer is needed
backend/benchmark/datasets/sglb_NN_<slug>.yaml        # generated by the builder
backend/data/benchmarks/sglb_NN_<slug>/{train,dev,test}.jsonl  # JSONL splits
backend/tests/test_sglb_NN_task.py                    # builder unit + scorer + oracle
Makefile                                              # + build-sglb-NN target
```

The existing tasks are reference implementations:

- SGLB-01 (`backend/data/ingestion/pdpc.py` + `backend/benchmark/tasks/sglb_01.py`):
  end-to-end ingestion + task; shows how to handle outcome-leakage redaction.
- SGLB-02 (`backend/benchmark/dataset_builders/sglb_02.py`): canonical
  builder shape — reads ingestion JSONL, applies filters, emits cases
  + harness YAML.
- SGLB-04 (`backend/benchmark/tasks/sglb_04.py`): smoke pattern; shows
  how a deterministic scorer can stand in as the oracle.

### 3. Required PR contents

- `make eval WORKFLOW=sglb_NN DATASET=backend/benchmark/datasets/sglb_NN_<slug>.yaml EVALUATORS="<your_scorers>"`
  returns 1.0 on the oracle.
- `pytest -x -q backend/tests/test_sglb_NN_task.py` passes.
- No new weak-tier evaluators (or, if absolutely needed, ship them with
  the `WEAK` tier flag — `--strict` will reject them by default).
- One paragraph in the spec's `## Limitations` documenting at least
  three falsifiable failure modes (cutoff drift, taxonomy completeness,
  sample skew, …).

### 4. Review path

- Two maintainer reviewers; one focuses on methodology, one on code.
- We will run your dataset through a frontier-model baseline before
  merging. If the model scores ≥98% on every split, the task is too
  easy and we ask for harder examples.
- We will run inter-annotator κ (LLM-judge ensemble) if your gold
  involves any LLM-judge step; κ < 0.4 sustained → rejected per
  coverage matrix §8.

## Adding errata

PR against `data/sglb_NN/errata.md` with one row per fix:

```
- id: sglb_NN_<hash>
  reason: gold citation "[2024] SGCA 5" should be "[2024] SGCA 5A"
  source_url: <link to the regulator/court page proving the fix>
  contributor: <github username>
```

Errata land at the next minor release (`v0.MAJOR.MINOR`) and bump the
dataset version on the affected task.

## Adding a public-source adapter

1. Create `backend/api/adapters/public/<source>.py` extending
   `LegalSourceAdapter` from `backend/api/adapters/base.py`.
2. Set `SourceMetadata.crawl_delay_seconds` to the source's documented
   rate-limit or 3s, whichever is larger.
3. Implement `fetch_all()` + `fetch_by_id()`; emit `SourceDocument`s
   with `extra_schema` fields that downstream task builders consume.
4. Add a `data.ingestion.<source>` CLI entry that writes a
   `vendor-data/<source>/<name>.jsonl` (one row per item, stable ID).
5. Test with a saved HTML fixture (`backend/tests/fixtures/<source>/`);
   we don't run live network requests in CI.
6. Register the adapter in `backend/api/adapters/__init__.py` if you
   want it exposed through the copilot.

**Hard rules for adapters:**

- No paywalled scrapers. Apply for the source's official partner API
  (SAL LawNet, LexisNexis SG) and document the route in your PR.
- No password-submission flows that impersonate the user against a
  commercial site.
- Respect `robots.txt` and any documented terms; cache aggressively to
  minimise upstream load.

## Repo conventions

- **Spec docs** live at `docs/sglb_specs/SGLB-NN.md`. The structure is
  enforced by `docs/sglb_specs/README.md`.
- **Code style** follows `backend/pyproject.toml` (ruff + mypy targets
  Python 3.11). Run `make lint` before pushing.
- **Test discipline:** every PR must add tests for its new code path.
  Use the existing `backend/tests/test_sglb_NN_task.py` files as
  templates. Mocking is acceptable for LLM clients; we never mock the
  scorers.
- **Synthetic data** lives in `*_candidates/` and is excluded from the
  reviewed corpus until explicitly promoted via `make synth-gen` →
  human review → `python -m benchmark.synthetic promote`. See
  `backend/benchmark/synthetic/README.md`.

## What we will not accept

- Tasks that score "subjective writing quality" (IRAC structure,
  persuasiveness) — coverage matrix §2 puts these out of scope.
- Tasks that require non-mechanical legal judgement at label time.
- Datasets sourced from material that isn't freely accessible (SLR
  full-text behind subscription, paid commentary, scraped private
  forum posts).
- "Beats GPT-X on legal tasks" framing in any artefact — see anti-
  snake-oil checklist (coverage matrix §5).

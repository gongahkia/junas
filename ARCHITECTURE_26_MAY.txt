What changed in this version vs ARCHITECTURE_25_MAY.txt
-------------------------------------------------------

1. /classify is no longer a "FROZEN legacy shim" — it's a thin wrapper over
   engine.review() returning a flat findings shape. Legacy layer1-6 + mosaic
   are DELETED from src/kaypoh/workflow/.

2. New planned signal layers (66 / 67 / 68) added as a side branch between LLM
   adjudication and source-verification. These REPLACE the conceptual role of
   the deleted layer1-6 stack with transparent, statute-citable additive
   signals — not a revival of the legacy code.

3. Fail-closed gate (item 50) extended into a workflow-wide roadmap item
   (item 65) covering container security, detector failures, LLM-tier failures,
   public-evidence retrieval failures, mapping-store failures, subject-erasure
   failures, citation-override resolution failures. Image-scan failures are now
   fail-closed, and /review, /anonymize, and /classify expose `degraded_modes`
   for the shipped OCR/redaction coverage gaps. The meta-audit is still open.

4. Image-scan / OCR layer (item 64) shipped between metadata scan and
   engine.review() — pluggable backend (Tesseract / OpenAI Vision / Google /
   AWS Rekognition / Azure). All cloud providers pass through PrivacyGuard,
   tenant-aware opt-in, diagnostics/readiness, confidence/box parsing where
   available, PDF page-render fallback, and best-effort redacted image artifacts
   on /anonymize.

5. Container coverage (item 61) substantially expanded: XLSX pivot caches,
   PPTX slide-masters, EML/MSG attachments recursive, ZIP/archive nesting,
   plus security vectors (password-protected, macro-enabled, polyglot, XXE,
   compression bombs, path traversal, embedded foreign docs).

6. Statutory anchors enumerated for every in-scope jurisdiction. HK / AU /
   JP / KR packs and one-fixture seed locks shipped under items 86 / 90, but
   remain seed-scale until item 92 expands coverage. Detector coverage maps
   to a specific statute, not a generic "we cover SG".


Key things this diagram makes obvious
-------------------------------------

• The deterministic engine IS the classifier. /classify is now a thin
  wrapper over engine.review(), not a separate pipeline.

• The LLM tier is a SIDE BRANCH, not the main path — only fires under
  audit_grade + ambiguous-score band + tenant opt-in.

• Items 66/67/68 are the conceptual heirs of the deleted legacy stack —
  classifier as additive recall booster, doc similarity as advisory
  signal, transparent aggregator replacing implicit max-severity-wins.
  None of these override the deterministic-high invariant.

• Everything privacy-sensitive (LLM, retrieval, image scan) passes
  through PrivacyGuard before any external call. Per-tenant opt-in is
  the standing principle for every new external-call feature.

• Fail-closed is a workflow-wide invariant, not just a PDF-ingest
  rule. Item 65 owns the remaining work to surface degraded_modes consistently
  when a layer can't fully execute.

• Three persistence stores have different durability semantics:
    - journal: append-only HMAC-chained
    - mapping store: mutable, Fernet-encrypt opt-in,
      reverse-index for subject erasure (planned)
    - session/matter store: mutable-by-design for inheritance

• Statute citations are per-jurisdiction and per-finding (item 85). The
  current override hook is KAYPOH_CITATIONS_OVERRIDE; item 60 moves override
  files to per-tenant resolution.
